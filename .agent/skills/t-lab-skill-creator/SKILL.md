# 🪄 T-Lab Skill Creator

「先生がテクノロジーで未来を変える」ための魔法の道具（スキル）を創るためのガイドです。
単なる自動化ツールではなく、先生の「余白（よはく）」を創り出し、子供たちと向き合う時間を生み出すための知恵を詰め込みます。

## 💡 T-Lab 理念の核

スキルを設計する際、以下の4つのシンボルを意識してください：

1. ⚡ **Innovation**: 「そんなことができるの！？」という驚きと新しさ。
2. ⚙️ **Efficiency**: 面倒な校務を秒で終わらせる、圧倒的なスピード。
3. 📖 **Knowledge**: 正しい知識とガイドラインに基づいた、信頼できる品質。
4. 🌱 **Growth**: 使うほどに先生が成長し、子供たちの未来が広がる設計。

## 🧙‍♂️ 「1/1000の魔法使い」へのアプローチ

T-Labのスキルは、単に「AIにやらせる」だけではありません。
- **仕組みを理解する**: GASやVibe Codingなど、自分でも作れるようになるヒントを。
- **分身（Avatar）を作る**: 先生の代わりに動き、24時間サポートするエージェントを。
- **共感と遊び心**: Harmonizerとして、使う人が楽しくなる、温かい言葉かけを。

```
project/

│   │       ├── SKILL.md     # 魔法の契約書（メインの指示）

│   │       └── assets/      # 魔法の素材（パーツ・素材）
```

## 🎨 T-Lab スタイル・ガイド

### 1. 遊び心を忘れない (Playfulness)
指示文や出力に、適度に絵文字やポジティブな言葉を混ぜましょう。
例：「修正しました」→「完璧に調整したよ！✨」

### 2. 「余白」を意識する (Yohaku)
そのスキルを使うことで、先生の時間がどれだけ浮くか、どんな価値ある時間に充てられるかを明示しましょう。

### 3. ステップ・バイ・ステップ (Growth)
難しいことを一気にやらせず、小さな成功体験を積み重ねられるワークフローを設計します。

## 🛠️ スキル作成プロセス

### Step 1: 魔法の種を見つける
- 「先生が一番困っていることは何？」
- 「どの作業が一番『余白』を奪っている？」
- 「どんなアウトプットがあれば、子供たちが喜ぶ？」

### Step 2: リソースの計画
- **scripts/**: 繰り返し発生する定型処理は、PythonやGASのコードに。
- **references/**: 文科省の最新ガイドラインや、T-Labの過去の知恵を。
- **assets/**: 素敵な通知表テンプレートや、スライドのボイラープレート。

### Step 3: スキルの初期化 (Run Command)
以下のコマンドで、T-Lab流のテンプレートからスキルを生成します。

```bash
python scripts/init_tlab_skill.py <skill-name> --path .agent/skills
```

### Step 4: 契約書 (SKILL.md) の編集
核心原則（T-Lab理念）を反映させながら、AIへの指示を書き込みます。

## ✅ 良いT-Labスキルのチェックリスト

- [ ] `description` に `Use when:` が明確にあるか？
- [ ] 500行以内に収まっているか？（詳細は `references/` へ）
- [ ] T-Labの4つのシンボルのうち、少なくとも1つが強調されているか？
- [ ] 使う人が「ワクワク」する遊び心が含まれているか？
- [ ] 出力形式はプレミアムで、そのまま使える美しさか？
